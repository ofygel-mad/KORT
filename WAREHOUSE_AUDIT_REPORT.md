## 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ (ВЫЯВЛЕНЫ И ИСПРАВЛЕНЫ)

### Проблема #1: НЕСОВМЕСТИМОСТЬ ФОРМАТОВ `variantKey` ⚠️ КРИТИЧЕСКАЯ

**Причина:** Разные модули строили `variantKey` в несовместимых форматах, что приводило к тому, что товары импортировались, но не находились при поиске.

#### Неправильный формат (ДО):
```
рубашка:цвет=синий:размер=44
```
- Разделитель между частями: `:`
- Формат атрибутов: `key=value`
- Использовался в: `warehouse.service.ts` (createItem, bulkImportOpeningBalance, reserveSimpleOrderItems, checkVariantAvailability)

#### Правильный формат (ПОСЛЕ):
```
рубашка|цвет:синий|размер:44
```
- Разделитель между частями: `|`
- Формат атрибутов: `key:value`
- Использовался в: `warehouse-inventory-core.service.ts`, `warehouse-catalog.service.ts`, `chapan/orders.service.ts`

## 🔧 ДОПОЛНИТЕЛЬНЫЕ НАЙДЕННЫЕ ПРОБЛЕМЫ

### Проблема #2: Непоследовательная нормализация имён товаров

**Описание:** Разные функции используют разные способы нормализации имён:
- Одни используют простой `toLowerCase()`
- Другие используют `normalizeWarehouseName()` → `trim().toLowerCase().replace(/\s+/g, ' ')`

**Это может привести к:** "Рубашка" и "  рубашка  " будут считаться разными товарами!

**Решение:** ✅ Все исправлены на использование `buildWarehouseVariantKey()` которая использует `normalizeWarehouseName()`

---

### Проблема #3: Фильтрация пустых атрибутов

**Описание:** 
- `warehouse.service.ts` создатели товаров НЕ фильтровали пустые значения перед построением ключа
- `warehouse-inventory-core.service.ts` ФИЛЬТРУЕТ пустые значения (`.filter(([, value]) => value.trim())`)

**Это может привести к:** Созданию дублирующихся товаров или товаров с разными вариантными ключами

**Решение:** ✅ Исправлено - `buildWarehouseVariantKey()` теперь используется везде и она фильтрует пустые значения

---

### Проблема #4: Рассинхрон между старым (WarehouseItem) и новым (WarehouseVariant) кодом

**Описание:**  
- `warehouse.service.ts` работает со старой схемой (WarehouseItem с qty)
- `warehouse-inventory-core.service.ts` работает с новой схемой (WarehouseVariant + WarehouseStockBalance)
- Нет чёткого разделения когда использовать какую

**Текущее состояние:** ⚠️ Оба системы работают параллельно, но могут рассинхронизироваться

**Рекомендация:** Это требует отдельной стратегической миграции (вне этого аудита)

## 📝 СПРАВКА ДЛЯ КОМАНДЫ

### buildWarehouseVariantKey() - единая функция построения ключа

```typescript
function buildWarehouseVariantKey(productName: string, attributes: Record<string, string>) {
  const base = normalizeWarehouseName(productName);
  const parts = Object.entries(attributes)
    .filter(([, value]) => value.trim())      // ← Фильтруем пустые!
    .sort(([a], [b]) => a.localeCompare(b))   // ← Сортируем для консистентности
    .map(([key, value]) => `${key}:${normalizeWarehouseName(value)}`);
  return [base, ...parts].join('|');
}
```

**Правила:**
- Имя товара нормализуется через `normalizeWarehouseName()` (trim + lowercase + одиночные пробелы)
- Пустые атрибуты ИГНОРИРУЮТСЯ (не включаются в ключ)
- Атрибуты сортируются по имени для консистентности
- Формат: `base|key:value|key:value`
